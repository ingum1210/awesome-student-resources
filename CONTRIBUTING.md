# Contributing to Awesome Student Resources

Thank you for your interest in contributing to Awesome Student Resources! This guide will help you add valuable resources that benefit students worldwide.

## Quick Contribution Guide

### Adding a New Resource

1. **Check if it already exists** - Search the README first
2. **Verify it's student-specific** - Must require .edu email or student verification
3. **Find the right category** - Choose from Cloud, Developer Tools, Design, Courses, etc.
4. **Add in alphabetical order** - Within the category table
5. **Follow the format** - Match existing entries exactly

### Format Template

```markdown
| Resource Name | Description of student benefits | https://direct-link-to-student-offer |
```

## Contribution Criteria

### What Qualifies

A resource must meet these requirements:

- **Student-specific offer** - Requires .edu email, student ID, or official verification
- **Free or heavily discounted** - Must be free OR 50%+ discount for students
- **Currently active** - Offer must be available at time of submission
- **Reputable source** - From legitimate, established companies or organizations
- **Genuine value** - Not just extended trials or minimal discounts

### What We Don't Accept

- Referral or affiliate links
- General free trials available to everyone
- Expired or discontinued offers
- Resources requiring upfront payment
- Duplicate entries

## How to Contribute

### Using GitHub (Recommended)

1. Fork this repository
2. Create a new branch (`git checkout -b add-resourcename`)
3. Edit `README.md` and add your resource to the appropriate category
4. Ensure alphabetical order within the category
5. Commit changes (`git commit -m 'Add ResourceName to Category'`)
6. Push to your fork (`git push origin add-resourcename`)
7. Open a Pull Request

### Using GitHub Web Interface

1. Click the pencil icon on `README.md`
2. Make your changes
3. Scroll down and click "Propose changes"
4. Click "Create pull request"

## Style Guidelines

### Description Format

Keep descriptions concise and objective:

**Good Examples:**
- `$100/year credits + 55+ always-free services`
- `Full access to all JetBrains IDEs during studies`
- `Premium plan with unlimited file uploads and advanced features`

**Avoid:**
- Subjective words: "amazing", "best", "incredible"
- Vague terms: "lots of free stuff", "great benefits"
- Marketing language: "revolutionary", "game-changing"

### Link Requirements

- Use direct links to the student offer page
- Include `https://` in all URLs
- No URL shorteners (bit.ly, tinyurl, etc.)
- No referral codes or tracking parameters
- Verify links work before submitting

## Pull Request Checklist

Before submitting your PR, ensure:

- [ ] Resource requires student verification (.edu email or student ID)
- [ ] Link is direct and working
- [ ] Entry is in alphabetical order within its category
- [ ] Description is clear and objective
- [ ] No duplicate entry exists
- [ ] Format matches existing entries
- [ ] No referral or affiliate links

## Example Pull Request

**Title:** Add Framer to Design Tools

**Description:**
```
Adding Framer for Students which offers free Pro access for students with .edu email verification.

Category: Design Tools
Link: https://www.framer.com/education/
Benefits: Pro access for building websites, portfolios & prototypes
```

## Review Process

1. A maintainer will review your PR within 2-3 days
2. If changes are needed, they'll be requested in the PR comments
3. Once approved, your contribution will be merged
4. You'll be credited as a contributor

- **Not sure about formatting?** Open an issue and ask
